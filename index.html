Product

The decision engine for travel booking.

Pigazzano sits between AI agents and OTAs. It aggregates availability and pricing from 16+ sources, applies a proprietary ranking model, and returns a single booking decision — not a list.

The problem

AI agents need decisions, not options.

When an AI agent needs to book a hotel, it can't present the user with 200 results and ask them to choose. It needs to pick one — confidently, immediately, correctly.

Today's travel APIs return lists. They return prices. They return data. None of them return a decision. That gap is what Pigazzano fills.

Without Pigazzano
Your agent calls 16 OTAs, receives thousands of results, implements its own ranking logic, and still guesses.
With Pigazzano
One API call. One hotel. One OTA. One confidence score. Ready to book.
How it works

Four layers. One answer.

1
Aggregation
We simultaneously query 16+ OTAs, normalizing prices, availability, and room types into a unified data model.
2
Enrichment
Each option is enriched with our proprietary signals: historical conversion rates, cancellation probability, price stability, and offer quality score.
3
Ranking
Our model ranks by booking success probability — not price alone. A cheaper option with a 45% cancellation rate scores lower than a slightly pricier one with 16%.
4
Decision
We return one recommended property with a confidence score, plus 2–3 alternatives. Your agent books.

Quickstart

Make your first decision call in under 5 minutes. No sandbox, no approval.

1
Create an account
Sign up at pigazzano.bluepillow.com. Your API key is generated automatically.
2
Get your API key
Find your key in Settings → API Keys.
PIGAZZANO_API_KEY=pk_live_xxxxxxxxxxxxxxxxxxxx
3
Make your first call
# curl
curl https://api.pigazzano.bluepillow.com/v1/decide \
-H "Authorization: Bearer $PIGAZZANO_API_KEY" \
-d '{"destination":"Rome","checkin":"2026-06-10","checkout":"2026-06-13","guests":2}'
4
Read the response
{
"recommended": {
"name": "Hotel de Russie",
"ota": "Booking.com",
"confidence": 0.94
},
"latency_ms": 118
}
That's it. Your agent now has a booking decision — no parsing, no ranking logic to implement.
Getting started

Authentication

All requests must include your API key in the Authorization header.

Authorization: Bearer pk_live_xxxxxxxxxxxxxxxxxxxx

Use pk_live_ for production and pk_test_ for test mode.

Never expose your API key in client-side code.
API reference

POST /decide

The core endpoint. Returns one booking decision with a confidence score.

POST/v1/decide
ParameterTypeDescription
destinationrequiredstringCity, region, or POI
checkinrequiredstringISO 8601
checkoutrequiredstringISO 8601
guestsrequiredintegerNumber of guests
budget_maxnumberMax price per night EUR
preferencesobjectstars, amenities, cancellation policy
Response
{
"recommended": { hotel_id, name, ota, price_eur, confidence },
"alternatives": [ ... ],
"latency_ms": 118
}
API reference

GET /alternatives

Returns the top 3 alternatives to a given decision.

GET/v1/alternatives
ParameterTypeDescription
decision_idrequiredstringID from /decide
excludearrayhotel_ids to exclude
