Product

The decision engine for travel booking.

Pigazzano sits between AI agents and OTAs. It aggregates availability and pricing from 16+ sources, applies a proprietary ranking model, and returns a single booking decision — not a list.

The problem

AI agents need decisions, not options.

When an AI agent needs to book a hotel, it can't present the user with 200 results and ask them to choose. It needs to pick one — confidently, immediately, correctly.

Today's travel APIs return lists. They return prices. They return data. None of them return a decision. That gap is what Pigazzano fills.

Without Pigazzano
Your agent calls 16 OTAs, receives thousands of results, implements its own ranking logic, and still guesses. Every team rebuilds the same wheel.
With Pigazzano
One API call. One hotel. One OTA. One confidence score. Ready to book. Your agent executes, it doesn't decide.
How it works

Four layers. One answer.

1
Aggregation
We simultaneously query 16+ OTAs — Booking.com, Agoda, Expedia, TripCom, and more — normalizing prices, availability, and room types into a unified data model.
2
Enrichment
Each option is enriched with our proprietary signals: historical conversion rates by country and property, cancellation probability, price stability, and offer quality score.
3
Ranking
Our model ranks by booking success probability — not by price alone. A cheaper option with a 45% cancellation rate scores lower than a slightly pricier one with 16%.
4
Decision
We return one recommended property with a confidence score, plus 2–3 alternatives. Your agent books. No parsing, no logic, no guesswork.

About

We're building the infrastructure layer travel was missing.

The travel industry has inventory. It has distribution. What it doesn't have is intelligence — a neutral layer that decides, not just lists. That's what we're building.

Where we come from

Blue Pillow has been operating in travel since 2017. Over the years we've processed over a million bookings, connected to 16+ OTAs, and built one of the most comprehensive lodging datasets in the industry — 7 million deduplicated properties, 285 million price combinations tracked daily, 1 TB of historical booking data.

We weren't planning to build a decision engine. We built it because we needed one ourselves. After years of routing bookings across OTAs, we realized the hardest problem wasn't finding options — it was picking the right one.

Quickstart

Make your first decision call in under 5 minutes. No sandbox, no approval — just sign up, get your API key, and start building.

1
Create an account
Sign up at pigazzano.bluepillow.com. Your API key is generated automatically.
2
Get your API key
Find your key in the dashboard under Settings → API Keys.
PIGAZZANO_API_KEY=pk_live_xxxxxxxxxxxxxxxxxxxx
3
Make your first call
Call POST /v1/decide with a destination and dates.
# curl
curl https://api.pigazzano.bluepillow.com/v1/decide \
-H "Authorization: Bearer $PIGAZZANO_API_KEY" \
-d '{"destination":"Rome","checkin":"2026-06-10","checkout":"2026-06-13","guests":2}'
4
Read the response

{

"confidence": 0.94

}

API reference

POST /decide

The core endpoint. Returns one booking decision with a confidence score.

POST/v1/decide
ParameterTypeDescription
destinationrequiredstringCity, region, or POI name
checkinrequiredstringCheck-in date ISO 8601
checkoutrequiredstringCheck-out date ISO 8601
guestsrequiredintegerNumber of guests
budget_maxnumberMax price per night in EUR
preferencesobjectOptional: stars, amenities, cancellation policy
Response
{
"recommended": { hotel_id, name, ota, price_eur, confidence },
"alternatives": [ ... ],
"metadata": { conversion_probability, cancellation_rate },
"latency_ms": 118
}
API reference

GET /alternatives

Returns the top 3 alternatives to a given decision.

GET/v1/alternatives
ParameterTypeDescription
decision_idrequiredstringID from a previous /decide call
excludearrayhotel_ids to exclude
API reference

POST /validate

Checks if a decision is still valid — price unchanged, availability confirmed.

POST/v1/validate
ParameterTypeDescription
decision_idrequiredstringThe decision ID to validate
Response
{
"valid": true,
"price_changed": false,
"available": true
}
